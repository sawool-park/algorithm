
특징

M           - 각 노드에서 최대 자녀 수
M-1         - 각 노드의 최대 Key 수
Ceil(M/2)   - 각 노드이 최소 자녀 노드 수 (root node, leaf node는 제외)
Ceil(M/2)-1 - 각 노드이 최소 Key 수 (root node 제외)

* Internal 노드의 Key 수가 X 개라면, 자녀 노드의 수는 언제나 X + 1 개다
* 노드가 최소 하나의 Key는 가지기 때문에 몇 차 B tree인지와 상관없이 internal 노드는 최소 두 개의 자녀는 갖는다
* M이 정해지면 root 노드를 제외하고 intenal 노드는 최소 Ceil(M/2) 개의 자녀 노드를 가질 수 있다.
  ex) M = 3이며 최소 자녀 노드 수는 2개이다.

삽입
* 추가는 항상 Leaf 노드에 한다
* 노드가 넘치면 가운데(median) Key를 기준으로 좌우 Key들을 분할하고 가운데 Key는 승진한다.
  - 노드의 최대 Key의 수 : M-1 개

삭제
* 삭제는 항상 Leaf 노드에 한다
* 삭제 후 최소 Key 수보다 작다면 재조정 한다
  - 최소 키 수 : Ceil(M/2) - 1
  - 최소 키 수의 제약은 Root 노드는 제외한다. 

    1. 해당 노드 검색한다.
    2. 노드의 Key 삭제한다
    3. 잔여 Key 수가 최소 키 수보다 크거나 같다면 종료한다.
        작다면, 4번 진행한다.
    4. Key 수 여유가 있는 형제(죄 -> 우 노드)의 지원을 받는다.
        부모 키를 내려 받고, 좌 또는 우측 노드 키를 올린다.
        - 죄측이면, 가장 큰 키를 부모의 가장 앞에 위치하도록 올린다.
        - 우측이면, 가장 작은 키를 부모의 가장 뒤에 위치하도록 올린다.
    5. 4번이 불가하다면, 부모의 지원을 받고 형제와 합친다.
        1) 부모의 키를 좌측으로 내리고 키를 정렬한다.
        2) 좌측 형제에 키를 합치고 노드를 지운다.
    6. 5번 후, 부모에 문제가 있다면, 거기서 다시 재조정한다.
